# Delete Node at Tail — Singly Linked List

## Conceptual Overview

In a **singly linked list**, every node stores its own data and a pointer referencing the next node in the sequence. The final node in the list is known as the **tail node**, and its `next` pointer always points to `None`, indicating the termination of the list.

Deleting the tail node means removing the **last element** of the linked list. Because a singly linked list does not maintain backward references, the algorithm must traverse the list until it finds the **second-last node**, also called the **penultimate node**. This node’s `next` pointer must then be updated to `None`, effectively detaching the tail node.

Example structure before deletion:

```
head
 ↓
10 → 20 → 30 → 40 → None
```

After deleting the tail node:

```
head
 ↓
10 → 20 → 30 → None
```

The node containing `40` becomes detached from the list.

---

# Algorithm

## Input

`head` — reference pointing to the first node of the linked list.

## Output

Updated linked list after removing the tail node.

## Algorithm Steps

1. Check whether the linked list is empty.
2. If the list contains no nodes, return `None`.
3. If the list contains only one node, deleting the tail removes the entire list, therefore return `None`.
4. Traverse the list until reaching the **second last node**.
5. The second last node satisfies the condition `current.next.next == None`.
6. Update the `next` pointer of the second last node to `None`.
7. Return the head reference because the head of the list does not change.

---

# Step-By-Step Execution

Assume the following linked list:

```
head
 ↓
10 → 20 → 30 → 40 → None
```

### Step 1 — Check for Empty List

```
head → None
```

If the head pointer is `None`, the list contains no nodes and deletion cannot occur.

---

### Step 2 — Handle Single Node Case

```
head
 ↓
10 → None
```

Deleting the tail here removes the only node in the list.

Result:

```
head → None
```

---

### Step 3 — Traverse to the Second Last Node

Traversal continues until the node whose `next.next` equals `None`.

Traversal process:

```
10 → 20 → 30 → 40 → None
            ↑
      second last node
```

---

### Step 4 — Remove the Tail Node

Update the pointer of the second last node.

```
current.next = None
```

Structure after update:

```
head
 ↓
10 → 20 → 30 → None
```

Node `40` becomes disconnected from the linked list.

---

# Python Implementation

```python
class Node:
    def __init__(self, data):
        # Stores the node value
        self.data = data
        
        # Reference to the next node in the list
        self.next = None


def delete_at_tail(head):
    """
    Deletes the last node of a singly linked list.

    The algorithm traverses the list until the second last
    node is located, then updates its next pointer to None.
    """

    # Case 1: Empty linked list
    if head is None:
        return None

    # Case 2: Only one node exists in the list
    if head.next is None:
        return None

    # Start traversal from the head node
    current = head

    # Move until reaching the second last node
    # Condition ensures current.next is the tail
    while current.next.next is not None:
        current = current.next

    # Disconnect the tail node
    current.next = None

    # Head remains unchanged
    return head
```

---

# Using the Function

```python
# Create nodes
node1 = Node(10)
node2 = Node(20)
node3 = Node(30)
node4 = Node(40)

# Create links
node1.next = node2
node2.next = node3
node3.next = node4

# Head pointer
head = node1

# Delete tail node
head = delete_at_tail(head)

# Traverse and print list
current = head
while current:
    print(current.data)
    current = current.next
```

---

# Expected Output

```
10
20
30
```

The node containing `40` has been removed from the linked list.

# Complexity Analysis

| Metric           | Value | Explanation                                  |
| ---------------- | ----- | -------------------------------------------- |
| Time Complexity  | O(n)  | Traversal required to reach second last node |
| Space Complexity | O(1)  | No additional memory allocation              |

The time complexity becomes linear because the algorithm must iterate through the linked list to locate the second last node before performing pointer reassignment.
