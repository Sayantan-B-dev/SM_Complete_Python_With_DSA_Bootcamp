# Insert at Index in a Singly Linked List (Recursive Approach)

## 1. Definition and Concept Overview

Insertion at a specific **index** using recursion involves progressively reducing the index while moving through the linked list until the target position is reached. Each recursive call processes one node and delegates the remainder of the operation to the next node.

The recursion stops when the required index becomes **0**, which represents the exact position where the new node must be inserted.

Example list before insertion:

```
10 -> 20 -> 30 -> None
```

Insert value `15` at index `1`.

Resulting list:

```
10 -> 15 -> 20 -> 30 -> None
```

Recursive traversal concept:

```
insert(head, index)
   |
   |-- insert(head.next, index-1)
           |
           |-- insertion occurs
```

Each recursive call moves one step forward in the list and decreases the index accordingly.

---

# 2. Core Principles and Internal Mechanics

Recursive insertion relies on two fundamental cases.

| Case           | Condition    | Action                              |
| -------------- | ------------ | ----------------------------------- |
| Base Case      | `index == 0` | Insert new node at current position |
| Recursive Case | `index > 0`  | Move to next node                   |

During recursion the structure of the linked list must remain intact, therefore pointer references must be reassigned correctly when returning from recursive calls.

Pointer update concept:

```
current -> next
```

After insertion:

```
current -> newNode -> next
```

---

# 3. Step-by-Step Logical Breakdown

### Step 1 — Handle Empty List

If the head pointer is `None` and insertion is requested at index `0`, the new node becomes the head.

---

### Step 2 — Base Case

When `index == 0`:

1. Create a new node.
2. Set its `next` pointer to the current node.
3. Return the new node so it becomes the new head of that sublist.

---

### Step 3 — Recursive Case

When `index > 0`:

1. Call the same function for the next node.
2. Reduce the index by one.
3. Reattach the returned node to maintain list structure.

---

# 4. Implementation (Python)

## Node Definition

```python
class Node:
    """
    Node structure for a singly linked list.
    Each node stores data and a pointer to the next node.
    """

    def __init__(self, value):

        # Store the data value
        self.data = value

        # Reference to the next node
        self.next = None
```

---

## Recursive Insert Function

```python
def insert_at_index_recursive(head, index, value):
    """
    Inserts a new node at a specific index using recursion.

    Parameters
    ----------
    head : Node
        Current node being processed

    index : int
        Target insertion index

    value : int
        Data value for the new node
    """

    # Base case: insert at this position
    if index == 0:

        # Create the new node
        new_node = Node(value)

        # Connect new node to current node
        new_node.next = head

        # Return new node as the updated head
        return new_node

    # If list ended before index reached
    if head is None:
        return None

    # Recursive call moving one node forward
    head.next = insert_at_index_recursive(head.next, index - 1, value)

    # Return current head to maintain connections
    return head
```

---

## Helper Function to Print List

```python
def print_list(head):
    """
    Traverses the linked list and prints node values.
    """

    temp = head

    while temp is not None:

        print(temp.data, end=" -> ")
        temp = temp.next

    print("None")
```

---

## Example Execution

```python
# Creating initial linked list
head = Node(10)
head.next = Node(20)
head.next.next = Node(30)

# Insert element at index 1
head = insert_at_index_recursive(head, 1, 15)

print_list(head)
```

---

## Expected Output

```
10 -> 15 -> 20 -> 30 -> None
```

# 5. Time and Space Complexity Analysis

### Time Complexity

| Operation          | Complexity |
| ------------------ | ---------- |
| Traversal to index | O(n)       |
| Node insertion     | O(1)       |

Total complexity:

```
O(n)
```

---

### Space Complexity

| Component        | Complexity |
| ---------------- | ---------- |
| Recursion stack  | O(n)       |
| Auxiliary memory | O(1)       |

Each recursive call consumes stack memory until the insertion point is reached.

---

# 6. Edge Cases and Failure Scenarios

### Insertion at Head

```
index = 0
```

The new node becomes the head of the list.

Example:

```
Before
10 -> 20 -> 30

After
5 -> 10 -> 20 -> 30
```

---

### Index Larger Than List Length

If recursion reaches `None` before index becomes `0`, insertion cannot be performed.

Example:

```
List: 10 -> 20 -> 30
Insert at index 10
```

Function returns the original list unchanged.

---

### Empty List

```
head = None
index = 0
```

Result:

```
newNode -> None
```

---

### Negative Index

Negative indices represent invalid operations and should typically be rejected before calling the recursive function.

# 8. Limitations and Trade-offs

| Limitation             | Explanation                                     |
| ---------------------- | ----------------------------------------------- |
| Stack memory usage     | Each recursive call consumes stack space        |
| Recursion depth limit  | Python recursion limit may restrict large lists |
| Function call overhead | Slightly slower than iterative approach         |

Iterative insertion avoids recursion stack overhead and is typically preferred for large-scale linked list operations.
