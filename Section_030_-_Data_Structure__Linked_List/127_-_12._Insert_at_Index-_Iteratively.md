# Insert at Index in a Singly Linked List (Iterative Method)

## 1. Definition and Concept Overview

Insertion at a specified **index** in a singly linked list means placing a new node at a particular positional offset relative to the head node. The structure must be traversed until the node immediately preceding the target position is located, after which pointer adjustments allow the new node to be integrated without breaking existing connections.

Indexing is assumed to follow **zero-based positioning**.

| Index | Node              |
| ----- | ----------------- |
| 0     | First node (head) |
| 1     | Second node       |
| 2     | Third node        |

Example initial list:

```
10 -> 20 -> 30 -> None
```

Insert `15` at index `1`.

Resulting structure:

```
10 -> 15 -> 20 -> 30 -> None
```

---

# 2. Core Principles and Internal Mechanics

Insertion requires maintaining the continuity of the linked structure.

| Operation            | Purpose                     |
| -------------------- | --------------------------- |
| Temporary pointer    | Used to traverse the list   |
| Counter variable     | Tracks traversal position   |
| Pointer reassignment | Updates links between nodes |

Pointer transformation during insertion:

```
Before insertion
prev -> current

After insertion
prev -> newNode -> current
```

Traversal must stop at **index − 1** so that the insertion occurs after the previous node.

---

# 3. Step-by-Step Logical Breakdown

### Step 1 — Create a Temporary Pointer

A traversal pointer (`temp`) is initialized with the head reference.

```
temp = head
```

This pointer moves across nodes without modifying the original head pointer.

---

### Step 2 — Traverse Until Index − 1

A counter variable tracks the traversal progress.

```
count = 0
```

The pointer moves until the node immediately before the insertion index is reached.

```
while count < index - 1:
    temp = temp.next
    count += 1
```

---

### Step 3 — Update Node Connections

Once the correct node is reached, pointer links are updated.

```
newNode.next = temp.next
temp.next = newNode
```

This preserves the remaining structure while inserting the new node.

---

# 4. Implementation (Python)

## Node Structure

```python
class Node:
    """
    Represents a node in a singly linked list.
    Stores data and a pointer to the next node.
    """

    def __init__(self, value):

        # Value stored in node
        self.data = value

        # Pointer to next node
        self.next = None
```

---

## Iterative Insert at Index

```python
def insert_at_index(head, index, value):
    """
    Inserts a node at the specified index using iteration.

    Steps implemented:
    1. Create traversal pointer
    2. Move pointer until index-1
    3. Update pointer connections
    """

    # Create node that will be inserted
    newNode = Node(value)

    # Case: insertion at head position
    if index == 0:

        # New node becomes head
        newNode.next = head
        return newNode

    # Step 1: create temporary traversal pointer
    temp = head

    # Counter used to track position
    count = 0

    # Step 2: move pointer until index - 1
    while temp is not None and count < index - 1:

        temp = temp.next
        count += 1

    # If index is larger than list length
    if temp is None:
        return head

    # Step 3: update node connections
    newNode.next = temp.next
    temp.next = newNode

    return head
```

---

## Helper Function for Printing

```python
def print_list(head):
    """
    Traverses the linked list and prints all node values.
    """

    temp = head

    while temp is not None:

        print(temp.data, end=" -> ")
        temp = temp.next

    print("None")
```

---

## Example Execution

```python
# Creating initial list
head = Node(10)
head.next = Node(20)
head.next.next = Node(30)

# Insert value at index
head = insert_at_index(head, 1, 15)

print_list(head)
```

10 -> 15 -> 20 -> 30 -> None

# 6. Edge Cases and Failure Scenarios

### Insertion at Head

```
index = 0
```

New node becomes the head of the list.

---

### Insertion Beyond List Length

Example:

```
10 -> 20 -> 30
Insert at index 10
```

Traversal reaches `None`. Insertion is ignored.

---

### Empty List

```
head = None
index = 0
```

Result:

```
newNode -> None
```

---

### Incorrect Pointer Update

Incorrect sequence:

```
temp.next = newNode
newNode.next = temp.next
```

This causes loss of the original next node reference.

Correct sequence:

```
newNode.next = temp.next
temp.next = newNode
```

| Graph algorithms    | Adjacency list modification       |

| Data stream systems | Ordered insertion operations      |

Insertion at arbitrary positions allows flexible manipulation of dynamically sized data structures.

---

# 8. Limitations and Trade-offs

| Limitation              | Explanation                          |
| ----------------------- | ------------------------------------ |
| Sequential traversal    | Required to reach insertion point    |
| Performance degradation | Large lists increase traversal time  |
| No random indexing      | Direct access by index is impossible |

Linked lists provide efficient pointer-based modifications but sacrifice the constant-time indexed access available in contiguous memory structures such as arrays.
