# Delete Node at Head — Singly Linked List

## Conceptual Overview

In a **singly linked list**, each node stores two pieces of information. The first part stores the **actual data**, while the second part stores a **reference to the next node** in the sequence. The list begins from a special reference called the **head pointer**, which always points to the first node of the list.

Deleting the node at the head means removing the **first element of the linked list**. Since the list is connected sequentially through `next` pointers, removing the head simply requires updating the head reference to point to the second node in the list.

Unlike array deletion, this operation does not require shifting elements. Only pointer reassignment is required, therefore the operation is extremely efficient.

Example representation before deletion:

```
head
 ↓
[10 | next] → [20 | next] → [30 | next] → None
```

After deleting the head node:

```
head
 ↓
[20 | next] → [30 | next] → None
```

The original first node containing `10` becomes detached from the structure.

---

# Algorithm

## Input

`head` — reference pointing to the first node of the linked list.

## Output

Updated head reference after removing the first node.

## Algorithm Steps

1. Check whether the linked list is empty.
2. If the list is empty, deletion cannot occur, therefore return `None`.
3. Store the node following the current head node.
4. This next node will become the new head of the linked list.
5. Optionally disconnect the original head node by setting its `next` reference to `None`.
6. Return the new head reference so that the caller can update the list pointer.

---

# Step-By-Step Execution

Assume the following linked list exists.

```
head
 ↓
10 → 20 → 30 → 40 → None
```

### Step 1 — Verify Whether List Is Empty

If the `head` pointer is `None`, the list contains no nodes.

```
head → None
```

In this case no deletion can occur.

---

### Step 2 — Identify New Head

The second node becomes the new starting point.

```
newHead = head.next
```

Structure during transition:

```
head
 ↓
10 → 20 → 30 → 40 → None

newHead
   ↓
  20
```

---

### Step 3 — Disconnect Old Head (Optional Safety Step)

```
head.next = None
```

This prevents accidental linkage or misuse of the removed node.

Structure becomes:

```
10 → None     20 → 30 → 40 → None
```

---

### Step 4 — Update Head Reference

Return the new head so that the calling code updates the linked list entry point.

```
head = newHead
```

Final structure:

```
head
 ↓
20 → 30 → 40 → None
```

---

# Python Implementation

```python
class Node:
    def __init__(self, data):
        # Node value storage
        self.data = data
        
        # Pointer to the next node in the list
        self.next = None


def delete_at_head(head):
    """
    Removes the first node of a singly linked list and returns
    the updated head reference.

    The function performs only pointer manipulation without
    traversing the linked list, therefore the operation runs
    in constant time.
    """

    # Step 1: Check whether the linked list is empty
    if head is None:
        # Nothing exists to delete
        return None

    # Step 2: Identify the next node which will become new head
    updated_head = head.next

    # Step 3: Disconnect the current head node
    # This step is optional but improves memory safety
    head.next = None

    # Step 4: Return the new head reference
    return updated_head
```

---

# Using the Function and Capturing Updated Head

```python
# Create nodes manually
node1 = Node(10)
node2 = Node(20)
node3 = Node(30)

# Create links between nodes
node1.next = node2
node2.next = node3

# Define head pointer
head = node1

# Perform deletion at head
head = delete_at_head(head)

# Traverse and print the updated linked list
current = head
while current:
    print(current.data)
    current = current.next
```

---

# Expected Output

```
20
30
```

The node containing `10` is removed, and the list now begins from `20`.

---

# Pointer State Transition

| Phase               | Head Pointer | Structure    |
| ------------------- | ------------ | ------------ |
| Initial state       | Node(10)     | 10 → 20 → 30 |
| New head identified | Node(10)     | newHead → 20 |
| Head reassigned     | Node(20)     | 20 → 30      |

---

# Complexity Analysis

| Metric           | Value | Explanation                              |
| ---------------- | ----- | ---------------------------------------- |
| Time Complexity  | O(1)  | Only pointer reassignment occurs         |
| Space Complexity | O(1)  | No additional memory allocation required |

Deletion at the head is the **fastest possible deletion operation in a linked list**, because the algorithm never needs to traverse the structure. Only the head pointer is modified.
